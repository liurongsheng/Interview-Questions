### RAIL测量模型
理想模型
- 响应：处理事件应在50ms以内完成
- 动画：每10ms产生一帧
- 空闲：尽可能增加空闲时间
- 加载：在5s内完成内容加载并可以交互

### 影响布局的操作

回流(reflow)：页面第一次加载完把内容放在页面上叫布局，之后页面发生变化后产生的再次布局就叫回流

- 添加或删除元素
- 操作styles
- display:none
- offsetLeft，scrollTop，clientWidth
- 移动元素位置
- 修改浏览器大小，文字大小

避免 layout thrashing (布局抖动)

根本原因是发生连续的强制回流，比如循环的更新部分元素的宽度的实现

- 避免回流
- 读写分离
可以是用[fastDom](https://github.com/wilsonpage/fastdom)插件批量对dom的读写操作 解决问题

减少重绘(repaint)

## webpack配置优化

Tree-shaking
- 上下文未用到的代码(dead code)
- 基于 ES6 import export
局限性，定义在 window对象中是无法用 import 中无法识别，所以需要在package.json中的sideEffects: [] 配置
- 
webpack4.0中如何启用，只要将 webpack 的 mode 设置为生产

babel 配置中 常使用 '@babel/preset-env' 在 配置属性中把 modules 设置为 false 来防止把 es6 的模块化语法转义掉

JS压缩
- Webpack4 后引入 uglifyjs-webpack-plugin
- 支持ES6替换为 terser-webpack-plugin，无论效率还是功能都比 uglifyjs 好，现在是生产的默认配置
- 减少 JS 文件体积

作用域提升，原理是把依赖关系进行合并，执行效率提升。需要注意 modules 的值需要为 false

### 提高构建效率
1. noParse
- 提高构建速度
- 直接通知 webpack 忽略较大的库
- 被忽略的库不能import，require，define引入方式
```
moudle: {
  noParse: /lodash/,
  rules: [
  ]
}
```

2. DllPlugin 把经常需要使用的库提取出来，变成引用的方式，开发环境使用比较经常
开始工程到最终上线都不会变的库，比如 react 和 react-dom 这两个库
- 避免打包时对不变的库重复构建
- 大大加快打包速度

### 代码拆分
- 把单个bundle文件拆分成若干的小bundles/chunks
- 缩短首屏加载时间
splitChunks 提取公有代码，拆分业务代码与第三方库
```
optimization:{
 splitChunks: {
  cacheGroups: {
   // 把第三方库抽取
    vendor: {
      name: 'vendor',
      test: /[\\/]node_modules[//\]/,
      minSize: 0,
      minChunks: 1,
      priority: 10,
      chunks: 'initial'
    },
    common: {
      name: 'common',
      test: /[//\]src[\\/]/,
      chunks: 'all',
    }
  }
 }
}
```
